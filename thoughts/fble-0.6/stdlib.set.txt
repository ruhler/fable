Set Library
===========
Closely related to map. What do I want?

What we have for Map@ today:
* Empty, IsEmpty, Put, Update, Remove, Replace, Get, ForEach

What we have for Set@ today:
* Empty, IsEmpty, Contains, Insert, Delete, ForEach

What I suggest changing Set@ to:
* Empty, IsEmpty, Add, Remove, Contains/Has?, ForEach

For List, I use Contains instead of Has. But for map I use 'Get'. And doesn't
'Has' sound nicer and it's shorter? I kind of vote for Has.

Contains means something else for list. It's the sublist. Not the element.
Otherwise we would use the Any function. But that doesn't do quite the same
for set, because it assumes linear search. So 'Has' makes sense to use.

What corresponds to Update and Replace? Update says add but combine values. We
don't have values to combine. So no need for it. Replace says maybe remove
depending on the value. But we don't have values, so again, no need.

Which leaves us with questions for bulk and combining operations.

The bulk operation would be a bulk remove. RemoveIf kind of thing.

The combining operations are maybe simpler now, because we don't have to merge
values in any way? Perhaps it is as simple as: And, Or, Sub?

And gives you the set of keys that in both. Or gives you the set that are in
either. Sub gives you the keys in the first but not the second. No combining
function. Maybe we just always take the key from the first set in case of
conflict, and you can turn them around the other way if you want the key in
the second set, except for Sub?

---

Proposal:
* And: Given two sets, returns the intersection of them.
  That is, all keys that appear in the first AND second map.
* Or: Given two sets, returns the union of them.
  All keys that appear in the first OR second map.
* Diff: Given sets A and B, returns all keys that appear in A but not B.
  This is set difference.
* Keep: Given a predicate of type (Key@) { Bool@; }, keeps all keys that
  satisfy the predicate.
* Omit: Given a predicate of type (Key@) { Bool@; }, removes all keys that
  satisfy the predicate.
  Uh... but Remove is overloaded. Hmm...
