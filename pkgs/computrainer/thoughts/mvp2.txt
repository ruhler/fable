MVP2
====
From MVP1 we have basic gates and running of the model.

MVP2 is about getting the gate editor UI into good enough state to be useful.

Proposal: Develop the editor sufficiently enough to be useful for implementing
a 4-bit adder, or a 4-bit counter.

Brainstorm of things to include:
* Fix graphics so you can tell what gate something is by looking at it
 - Need to use color to distinguish between, e.g. 0 and 1 output of switch
   gate.
* Fix model to advance gates in addition to engines, so we don't have to worry
  about extra space requirements outside of 0 and 1 gates.
* Fundamental editing features
 - Place any gate of my choice anywhere, easily.
 - Remove gates. Modify existing gates: rotate, change type.
 - Place/remove/edit cars on the map.
 - Undo?
* Pause / resume execution of the model.
* Load and save models/cars to file.

Don't worry about modularization yet. One flat model that I can edit and
expand and save between sessions.

Let's say you pass the name of a model file at startup, you have some way to
explicitly 'save' to file. Or maybe it auto-saves when you close.

Question: Should we support highlighting a rectangular region and copying it
and pasting it somewhere else? How about yes if it turns out to be useful
(which it almost certainly will).

Sounds like a plan.
