Input methods:
* keyboard only
* keyboard + mouse
* mouse only
* touch screen only

Ideally we could easily switch between all the different input methods, and
there is some consistency between them. Touch screen is hardest. Maybe don't
worry about that to start.

I think keyboard + mouse idea is keyboard is the left hand, mouse is the right
hand. So right hand keyboard and mouse should ideally cover the same
operations.

Typical mouse operations: zoom, select, place.

A lot of the mouse only stuff I envision being a pallet of buttons you can
click with the mouse to have the same effect a keyboard button would give you.

Big next thing to make building networks practical: placing gates.

Here's how I envision things:
* We have a pallet of gate options, probably at the bottom of the screen.
* You can always see exact pixel location of cursor.
* When you click on a gate option, show a phantom gate under the cursor
  aligned to the grid. You can still see exact pixel location of cursor.
* Press 'R' to rotate the phantom gate. Press 'v' or 'h' to flip the phantom
  gate. Press some other key to cycle through other config variations.
* Mouse down to make phantom gate real.
  - Let's say it replaces the gate beneath it, if any.
  - You keep a phantom gate under the cursor.
* Some way to show conflict when hovering phantom gate over an existing gate?
* Press escape or equivalent key to drop the phantom gate.
* If you drag and hold with the mouse down, any tile the mouse enters gets the
  phantom gate applied to it.
* You can click on an existing gate if you don't have a phantom under the
  cursor. That selects the gate. Then clicking 'R', 'v', 'h', etc. will take
  direct effect on that gate, without additional clicking required.
* Press escape or equivalent key to deselect an existing selected gate.
* Perhaps we assign key bindings to each kind of gate. Then if you press that
  key, it adds a phantom to an otherwise empty cursor, changes the phantom on
  a phantom cursor, or changes the placed gate on a selected cursor. Ideally
  it keeps the same rotation direction as was already in place, if any.
* Press delete key on a select gate to remove that gate. Cursor becomes empty.

Basically we have three states for the cursor:
* Empty
* Phantom
* Selected

Sounds pretty complicated. But also fairly well defined behavior?
