Here's my vote:

The goal for the hello world tutorial is to make it easy for the reader to
play around with different fble code and see the effect. A single
Stream@<Char@> output is fine for that. No need for command line arguments or
stdin, because they can put whatever inputs they want directly in the .fble
file. Simple, yet functional.

I do want to show how to do multiple puts with Do, and maybe Return within the
Stream@<Char@> definition. That can be the second step. Then we can do format
strings. Then we can show different types.

I feel like my tutorial is good for deep dive exhaustive coverage of the
language. But it's maybe not so good for getting started writing programs. If
we wanted to get started writing programs, I would focus more on those things
that are more common, ignore things that are esoteric, and provide a lot more
about things like how to set up your own build system to compile fble code. I
would demonstrate how to write things using the standard library rather than
go out of my way to have people write things without the standard library.

Like, you write code differently if writing an end user application with the
standard library than you would if writing the standard library yourself.
